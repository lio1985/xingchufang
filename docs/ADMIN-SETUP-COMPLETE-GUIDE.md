# 管理员设置完整指南

## 一、管理员类型说明

### 1. 超级管理员
- **自动激活**：登录时自动获得管理员权限，无需等待审核
- **配置方式**：通过环境变量 `SUPER_ADMIN_OPENID` 配置
- **权限**：拥有所有管理员权限
- **用途**：用于第一个管理员设置

### 2. 普通管理员
- **需要审核**：由超级管理员或其他管理员授权
- **设置方式**：通过管理后台或脚本设置
- **权限**：拥有管理员权限
- **用途**：用于团队协作管理

## 二、设置超级管理员（推荐）

### 方式1：环境变量配置（推荐）

这是最简单的方式，适合设置第一个管理员。

#### 步骤1：登录小程序获取openid

1. 使用你的微信账号登录小程序
2. 点击"微信一键登录"
3. 登录成功后，你的账号已被创建

#### 步骤2：获取openid

在服务器上运行以下命令：

```bash
cd server
node scripts/get-user-list.js
```

输出示例：
```

├────────────────────────────────────────┼────────────────────────────────────────┤
│ 123e4567-e89b-12d3-a456-426614174000 │ 用户abc123                             │
└────────────────────────────────────────┴────────────────────────────────────────┘

用户详情:
- ID: 123e4567-e89b-12d3-a456-426614174000
- OpenID: o6_bmasdasdsad6_2sgVt7hMZOPfL

- 角色: user
- 状态: pending
```

复制你的 `OpenID`。

#### 步骤3：配置环境变量

编辑 `.env.local` 文件，添加超级管理员openid：

```bash
# ===================================
# 超级管理员配置 (可选)
# ===================================
SUPER_ADMIN_OPENID=o6_bmasdasdsad6_2sgVt7hMZOPfL
```

#### 步骤4：重启后端服务

```bash
# 如果使用 coze dev
cd /workspace/projects
coze dev

# 或者重启后端服务
cd server
npm run build
pm2 restart server
```

#### 步骤5：重新登录

使用你的微信账号重新登录小程序，系统会自动识别你是超级管理员：
- 角色自动设置为 `admin`
- 状态自动设置为 `active`
- 无需等待审核，可以直接使用管理后台

## 三、添加其他管理员

已经有超级管理员后，可以通过管理后台添加其他管理员。

### 方式1：管理后台设置（推荐）

1. 使用超级管理员账号登录小程序
2. 进入管理后台 > 用户管理
3. 找到要设置为管理员的用户
4. 点击用户详情
5. 将角色改为"admin"
6. 点击保存

### 方式2：脚本设置

```bash
cd server
node scripts/setup-admin.js <目标用户的openid>
```

## 四、管理员权限说明

### 管理员可以做什么：

1. **用户管理**
   - 查看所有用户
   - 启用/禁用用户
   - 设置用户角色（普通用户/管理员）
   - 查看用户详情和统计

2. **内容审核**
   - 审核待审核用户
   - 查看用户操作日志
   - 管理语料库共享

3. **数据管理**
   - 查看全局统计数据
   - 导出用户数据
   - 查看智能运营报告

4. **系统管理**
   - 配置语料库全局共享
   - 查看共享统计
   - 管理快速笔记

### 普通用户权限：

1. 使用所有创作功能（选题生成、内容生成、语料优化、爆款复刻）
2. 管理自己的语料库
3. 查看自己的统计数据
4. 使用快速笔记功能

## 五、常见问题

### Q1: 配置了SUPER_ADMIN_OPENID但不是管理员？

**A**: 可能的原因：
- openid配置错误，请仔细核对
- 配置后未重启后端服务，请重启服务
- 使用了旧的token，请重新登录

**解决方法**：
```bash
# 检查openid是否正确
node server/scripts/get-user-list.js

# 重启后端服务
coze dev

# 重新登录小程序，清除旧token
```

### Q2: 如何查看当前的管理员列表？

**A**:
```bash
cd server
node scripts/get-user-list.js

# 查找 role 为 admin 的用户
```

或者在管理后台 > 用户管理中筛选角色为"管理员"。

### Q3: 如何取消管理员权限？

**A**:
1. 进入管理后台 > 用户管理
2. 找到要取消权限的管理员
3. 点击用户详情
4. 将角色改为"user"
5. 点击保存

### Q4: 超级管理员和普通管理员有什么区别？

**A**:
- **超级管理员**：登录时自动激活，无需审核
- **普通管理员**：需要超级管理员授权，首次登录需等待审核

两者在权限上没有区别，都拥有管理员权限。

### Q5: 可以有多个超级管理员吗？

**A**:
技术上可以，但不建议。`SUPER_ADMIN_OPENID` 只能配置一个openid。

如果需要多个管理员，建议：
- 配置一个超级管理员
- 通过管理后台添加其他管理员

### Q6: 忘记配置超级管理员怎么办？

**A**:
使用方式2（脚本设置）手动设置第一个管理员。

### Q7: 如何修改超级管理员？

**A**:
1. 修改 `.env.local` 文件中的 `SUPER_ADMIN_OPENID`
2. 重启后端服务
3. 新超级管理员登录时自动获得管理员权限
4. 可选：取消旧超级管理员的管理员权限

### Q8: 测试环境如何配置超级管理员？

**A**:
开发环境可以直接使用模拟openid：

```bash
# .env.local
NODE_ENV=development
SUPER_ADMIN_OPENID=mock_openid_test
```

使用模拟openid登录时：
```javascript
openid = `mock_openid_${code}`;
```

如果 `code` 是 "test"，则 openid 为 "mock_openid_test"。

## 六、安全建议

### 1. 保护 SUPER_ADMIN_OPENID
- 不要将包含真实openid的 `.env.local` 提交到Git仓库
- 生产环境使用密钥管理系统存储环境变量
- 定期更换超级管理员（可选）

### 2. 最小权限原则
- 不要给不必要的用户管理员权限
- 定期审查管理员列表
- 员工离职时及时取消管理员权限

### 3. 审计日志
- 定期查看用户操作日志
- 关注管理员的操作记录
- 发现异常及时处理

### 4. 定期备份
- 定期备份数据库
- 备份管理员配置信息
- 制定恢复计划

## 八、技术说明

### 超级管理员自动激活原理

```typescript
// server/src/user/user.service.ts

// 检查是否是超级管理员
const superAdminOpenid = process.env.SUPER_ADMIN_OPENID;
const isSuperAdmin = superAdminOpenid && openid === superAdminOpenid;

// 创建新用户
const newUser = {
  openid,
  unionid,
  nickname: `用户${Math.random().toString(36).substr(2, 6)}`,
  role: isSuperAdmin ? 'admin' : 'user',       // 超级管理员自动设置为admin
  status: isSuperAdmin ? 'active' : 'pending', // 超级管理员直接激活
  created_at: now,
  updated_at: now,
};
```

## 十、总结

通过超级管理员机制，您可以轻松设置第一个管理员：

✅ **简单**：配置一个环境变量即可
✅ **自动**：登录时自动激活，无需手动设置
✅ **灵活**：还可以通过脚本或管理后台添加其他管理员
✅ **安全**：基于openid验证，确保身份真实
✅ **可控**：可以随时取消管理员权限

建议流程：
1. 使用环境变量配置超级管理员（第一个管理员）
2. 通过管理后台添加其他管理员（团队协作）
3. 定期审查管理员列表
4. 及时离职员工权限
