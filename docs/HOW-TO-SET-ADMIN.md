# 🎯 如何设置小程序管理员

## 问题说明

发布小程序到微信后，通过微信一键登录，系统如何知道你是管理员？

---

## 💡 解决方案（2 步设置）

### 步骤 1：在小程序中查看你的 OpenID

1. **登录小程序**
   - 使用你的微信账号登录小程序
   - 点击「微信一键登录」

2. **查看用户信息**
   - 登录成功后，在首页右上角找到「用户信息」按钮（蓝色文档图标）
   - 点击后会弹出对话框，显示你的用户信息
   - 找到「OpenID」字段，这就是你的微信唯一标识

3. **复制 OpenID**
   - 点击对话框底部的「复制 OpenID」按钮
   - OpenID 已复制到剪贴板

---

### 步骤 2：将 OpenID 配置到服务器

#### 方法 1：通过环境变量配置（推荐）

1. 登录服务器
2. 编辑环境变量文件：
   ```bash
   cd /opt/xingchufang
   nano .env
   ```

3. 找到 `SUPER_ADMIN_OPENID` 配置项，填入你的 OpenID：
   ```bash
   SUPER_ADMIN_OPENID=oP9lR5f8wX2hK6mJ3qY7tN4sZ1v
   ```

4. 保存并退出（Ctrl + X，然后 Y，然后 Enter）

5. 重启后端服务：
   ```bash
   pm2 restart xingchufang-server
   ```

6. 重新登录小程序，你将成为管理员

#### 方法 2：通过数据库直接修改

1. 登录 Supabase 控制台
2. 进入表编辑器（Table Editor）
3. 选择 `users` 表
4. 找到你的用户记录（通过昵称或 OpenID 搜索）
5. 将 `role` 字段修改为 `admin`
6. 重新登录小程序，你将成为管理员

## 🎨 用户信息对话框说明

### 显示内容

| 字段 | 说明 |
|------|------|
| 昵称 | 你的用户昵称 |
| OpenID | 微信唯一标识（用于配置管理员） |
| 角色 | 当前角色（管理员 / 普通用户） |
| 状态 | 账号状态（正常 / 待审核 / 已禁用 / 已删除） |
| 用户 ID | 系统内部用户 ID |

### 操作按钮

- **✕ 关闭**：关闭对话框
- **复制 OpenID**：将 OpenID 复制到剪贴板

---

## 🔍 如何确认是否是管理员？

### 方法 1：查看用户信息对话框

- 登录小程序
- 点击「用户信息」按钮
- 查看「角色」字段是否显示「管理员」

### 方法 2：查看首页是否有管理员入口

- 登录小程序
- 查看首页右上角是否有「管理员入口」按钮（绿色盾牌图标）
- 如果有，说明你已经是管理员

---

## ⚠️ 注意事项

1. **OpenID 唯一性**：
   - 每个微信用户在不同小程序中有不同的 OpenID
   - OpenID 是微信为你分配的唯一标识，不会重复

2. **配置后需重启**：
   - 修改 `.env` 文件后，必须重启后端服务才能生效
   - 命令：`pm2 restart xingchufang-server`

3. **多个管理员**：
   - 如果需要多个管理员，需要重复以上步骤
   - 或者通过管理后台「用户管理」页面设置管理员

4. **数据库直接修改**：
   - 直接修改数据库后，不需要重启服务，重新登录即可生效

## ❓ 常见问题

### Q: 为什么登录后看不到用户信息按钮？

**A**: 请检查：
- 是否已成功登录
- 是否在首页（返回首页查看）
- 是否有多个版本，确认使用最新版本

### Q: 复制 OpenID 后，配置了但还是管理员？

**A**: 请检查：
- 是否重启了后端服务
- OpenID 是否正确复制（不要有空格）
- `.env` 文件是否正确保存

### Q: 如何设置其他用户为管理员？

**A**:
- 让其他用户登录小程序，获取他们的 OpenID
- 将 OpenID 添加到环境变量（需要支持多个 OpenID）
- 或者通过管理后台「用户管理」页面设置

### Q: H5 环境还需要配置 OpenID 吗？

**A**:
- 不需要，H5 环境所有模拟登录的用户自动成为管理员
- 这是用于测试的临时方案
