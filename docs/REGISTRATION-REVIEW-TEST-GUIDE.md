# 注册审核流程测试指南

## 功能说明

本小程序采用"注册后待审核"机制：
1. 任何用户都可以点击"微信一键登录"
2. 首次登录时，系统自动创建账号，状态为"pending"（待审核）
3. 待审核用户可以进入首页，但无法使用任何功能
4. 管理员在后台审核用户，将状态改为"active"后，用户才能正常使用

## 测试流程

### 前置条件

1. **管理员账号**
   - 需要有一个管理员账号
   - 可以使用 `server/scripts/setup-admin.js` 脚本设置管理员

2. **测试用户**
   - 需要至少2个微信账号（一个用于测试新用户注册，一个用于管理员）

### 测试步骤

#### 步骤1：设置管理员

```bash
cd server
node scripts/get-user-list.js
```

查看用户列表，获取管理员的openid，然后设置管理员：

```bash
node scripts/setup-admin.js <管理员openid>
```

#### 步骤2：测试新用户注册

1. 使用另一个微信账号登录小程序
2. 点击"微信一键登录"按钮
3. 登录成功后，应该看到"等待管理员审核"的提示
4. 进入首页，可以看到顶部的"等待审核"状态卡片
5. 点击任何功能卡片，应该提示"您的账号正在等待管理员审核"

#### 步骤3：管理员审核用户

1. 使用管理员账号登录小程序
2. 进入管理后台 > 用户管理
3. 在状态筛选中选择"待审核"
4. 找到刚才注册的新用户
5. 点击用户详情
6. 将用户状态改为"正常"（active）

#### 步骤4：测试用户正常使用

1. 使用新用户账号重新登录小程序
2. 登录成功后，应该直接进入首页，没有"等待审核"提示
3. 可以正常使用所有功能

### 预期结果

**场景1：新用户首次登录**
- ✅ 登录成功
- ✅ 显示"等待管理员审核"提示
- ✅ 进入首页，显示"等待审核"状态卡片
- ✅ 点击功能卡片时提示"等待审核"
- ✅ 无法使用任何功能

**场景2：管理员审核通过**
- ✅ 管理员可以在用户列表中看到待审核用户
- ✅ 管理员可以修改用户状态为"正常"
- ✅ 状态修改成功

**场景3：用户审核通过后登录**
- ✅ 登录成功，没有"等待审核"提示
- ✅ 可以正常使用所有功能

## 代码修改说明

### 后端修改

**1. 用户登录逻辑**（`server/src/user/user.service.ts`）
```typescript
// 创建新用户时，默认状态改为'pending'
status: 'pending', // 默认为 pending，等待管理员审核
```

**2. 管理后台接口**（`server/src/admin/admin.controller.ts`）
```typescript
// 支持修改用户状态为'pending'
@Put('users/:userId/status')
async updateUserStatus(
  @Body() body: { status: 'active' | 'disabled' | 'deleted' | 'pending' }
)
```

### 前端修改

**1. 登录页面**（`src/pages/login/index.tsx`）
- 根据用户状态显示不同提示
- pending状态：显示"等待管理员审核"对话框
- disabled/deleted状态：显示"账号已被禁用"对话框

**2. 首页**（`src/pages/index/index.tsx`）
- 添加用户状态判断
- pending状态：显示"等待审核"状态卡片
- pending状态：阻止功能使用，点击时提示"等待审核"

**3. 管理后台用户管理**（`src/pages/admin/users/index.tsx`）
- 添加"待审核"状态筛选
- 添加"待审核"状态标签显示

## 数据库影响

**users表**
- status字段支持：'active' | 'pending' | 'disabled' | 'deleted'
- 默认状态：'pending'（新增用户）

## 注意事项

1. **现有用户不受影响**
   - 之前创建的用户状态为'active'，不受影响
   - 可以正常使用

2. **管理员账号不受限制**
   - 管理员账号始终可以使用所有功能
   - 无论什么状态

3. **审核流程建议**
   - 建议定期查看待审核用户列表
   - 及时审核新用户，提升用户体验
   - 可以根据实际情况调整审核标准

## 扩展功能建议

### 1. 批量审核功能
管理员可以批量选择多个待审核用户，一键批准或拒绝。

### 2. 审核通知功能
用户审核通过后，发送微信模板消息通知用户。

### 3. 审核备注功能
管理员在审核时可以添加备注，记录审核理由。

### 4. 自动审核规则
可以根据用户信息自动判断是否通过审核：
- 微信昵称包含公司关键词
- 微信手机号在公司通讯录中

## 问题排查

### 问题1：登录后没有提示"等待审核"
**可能原因：**
- 用户状态已经是'active'（旧用户）
- 代码未正确部署

**解决方法：**
- 检查数据库中用户状态
- 清除缓存重新登录

### 问题2：管理员无法看到待审核用户
**可能原因：**
- 后端代码未更新
- 状态筛选功能有问题

**解决方法：**
- 确认后端代码已更新并重启
- 检查网络请求是否正常

### 问题3：待审核用户仍可使用功能
**可能原因：**
- 前端状态判断有问题
- 后端接口未验证用户状态

**解决方法：**
- 检查前端handleNavigateTo函数
- 在后端接口中添加状态验证
