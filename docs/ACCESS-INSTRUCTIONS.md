### ⚠️ 方式 2：在本地浏览器访问 localhost（不适用）

## 🔍 故障排查

### 问题：浏览器显示"无法访问此网站"

**可能原因**：
1. 访问的地址不对
2. 端口被防火墙阻止
3. 服务没有正常启动

**排查步骤**：

1. **检查服务是否运行**：
   ```bash
   ss -tuln | grep 5000
   ```
   应该看到：
   ```
   tcp   LISTEN 0      511    0.0.0.0:5000    0.0.0.0:*
   ```

2. **检查 HTTP 响应**：
   ```bash
   curl -I http://localhost:5000
   ```
   应该返回：
   ```
   HTTP/1.1 200 OK
   ```

3. **查看服务日志**：
   ```bash
   tail -20 /tmp/coze-logs/dev.log
   ```

## 🆘 常见问题

### Q: 我在本地访问 localhost:5000，但是显示"无法连接"

**A**: 检查以下几点：
1. 确认服务在运行：`ps aux | grep taro`
2. 检查端口监听：`ss -tuln | grep 5000`
3. 重启服务：`cd /workspace/projects && coze dev`

### Q: 我在 Coze 环境中，但是域名打不开

**A**: 可能需要配置反向代理。当前服务在 localhost:5000 运行，需要将 Coze 域名代理到这个地址。

### Q: 如何确认服务是否正常？

**A**: 使用以下命令测试：
```bash
curl http://localhost:5000
```
如果返回 HTML 内容，说明服务正常。
